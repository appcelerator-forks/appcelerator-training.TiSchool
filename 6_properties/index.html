What are Properties?

Properties

  • Simple storage mechanism
  • Limited capacity
  • Fast & flexible
Properties are a simple storage mechanism in which you can store what are called key/value pairs. Basically, you can store a variable (that's the key) and its contents are the value. You should consider properties to have a limited capacity. We'll talk more about why later. But you won't store all your app's data in properties. They're convenient because they're quick to create & retrieve and can store just about any type of data you want to put in them.

Storage & Retrieval

Creating an app property


Ti.App.Properties.setString('key', 'value');

// for example
Ti.App.Properties.setString('playerName', 'Gandalf');

Retrieving an app property


Ti.App.Properties.getString('key', 'default value');

// for example
var player = Ti.App.Properties.getString('playerName'); // second arg is optional

Caveat!

  • Read into memory when app starts
  • Stay there till app closes
  • Memory for apps is very limited!
  • Don't store a lot in properties
