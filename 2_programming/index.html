Programming Steps

  1. Write your source code
  2. Compile / build
  3. Deploy
  4. Test
Explain what happens at each step.

Note: Most times, you don't compile JavaScript. However, there are similar steps. In a browser environment, you might minimize and obfuscate your code. In Titanium, you do compile your app, which combines your JavaScript code with the Titanium-provided components and calls the native compilation tools.

Let's be clear ... computers are dumb

  • Computers will do exactly what you say
  • They don't interpret, think, guess, or read between the lines
  • They'll do something as many times as you say (without complaining)
  • When you stop telling them what to do, they stop doing anything
So, of the preceding programming steps, testing is perhaps the most important. In the real world, a company often employs many people to test their software. Then they'll ask customers or key partners to futher test the software before it's finally released to the general user base.

JavaScript

So, what are our building blocks?

  • Data
  • Variables
  • Statements ("commands")
  • Conditionals & Control Statements

You should have completed the pre-requisite exercises. So, you should know what all these constructs are and how you'd use them. We'll go through them quickly as review and to make sure everyone has at least a basic understanding of JavaScript

Data Types


1 // integer (whole numbers)
1.23456 // float (fractions)
true // or false == boolean
['one', 'two', 'three'] // array
{ color: 'blue', size: 10 } // object
null, undefined, NaN, Infinity // special types
Because JavaScript is type-flexible, our biggest concern here is learning the terminology and understanding the complex data types: arrays and objects, plus the special types.

Variables


var x = 1;
var y = 'foo'; // enclose strings in quotes
var i = 0, j = 1, k = true; // declare multiple w/single statement
y = 4; // reassigns y's value
x = y; // put y's value into x, so x = ?
  • Always declare using var keyword
  • JavaScript doesn't care about data types (but you should)
Explain each of the examples. While var isn't strictly required, treat it as if it were. At this stage, students probably aren't ready for a discussion of variable scope and the subtlety that omitting var puts the variable into the global scope. Our suggestion is that you impress on them that using 'var' is critical (even if not technically required).

For Loops

While Loops


while(somecondition) {
	// do something till somecondition is no longer met
}
  • Make sure 'somecondition' will change and eventually be met or loop will run forever

Switch


switch(variable) {
  case value1:
     // do something if variable == value1
  break;
  case value2:
     // do something if variable == value2
  break;
  default:
     // do something if none of the above are met
  break;
}
  • Compact if/else statement

Functions

  // do as many statements as needed
  return a + b;
}
// then use like this:

  • Reusable chunks of code
  • Arguments pass data to the function

Function Statements


var sum = function(a, b) {
  return a + b;
}
// then use like this:
var total = sum(100, 200);
  • Equivalent to other format
  • You're defining a variable that will hold a function
  • Highlights that functions are 'first class citizens' in JavaScript
  • You can pass a function as an argument to another function

Function Callbacks


var doCoolness = function(a, b, fn) {
  // something here that takes a long time to finish, such
  // as getting data from the network
  return fn(a, b);
}
var sum = function(a, b) {
  return a + b;
}
// then use like this:
var total = doCoolness(100, 200, sum);
  • Great when doCoolness() might take a long time to run (like when accessing the network).
Callbacks are a critical part of modern JavaScript programming. They are the primary pattern for dealing with asynchronous operations, such as network access, geolocation lookups, even database operations.

JavaScript in Pieces

  • Core JavaScript
    • Core operators
    • Objects like Date and Math
    • Overall language rules & syntax
  • Host environment adds its own objects
    • Web browser (DOM)
    • Node.js (server-side functionality)
    • Titanium (Ti object for mobile components)
  • Libraries you import (APIs)
JavaScript is implemented in various environments. And the features and functionality it offers vary depending on the environment. The core JavaScript features are the same everywhere. This includes core operators, statements, and syntax, plus the core objects like Date and Math. The host environment adds new objects. For example, JavaScript in the browser has access to objects that represent the web page and web browser. Titanium adds the Ti object to core JavaScript, giving you access to mobile features. But in Titanium, you don't have access to the DOM objects.

Titanium Introduction

How Titanium Works

  • You write Javascript
  • Titanium maps your JavaScript code to native components
  • Communication is two-way across the bridge

The key points to understand are:
  • You write your mobile app in JavaScript
  • When you create something, like a button, in JavaScript, Titanium creates a native equivalent for you
  • When you modify the button (set it's title or color), Titanium maps that change to native land
  • When the user clicks the button, your JavaScript code can react

Titanium Key Points:

  • You write in JavaScript
  • There's a platform-specific bridge and set of native components
    (so your source code can run on multiple platforms)
  • Your JavaScript is still there and running on the user's device
  • Your app is compiled (so your code is secure)
  • But you still can't ignore platform differences

Titanium Provides APIs For:

  • UI components — add native buttons, pickers, tables and more to your apps
  • Networking — HTTP/HTTPS, sockets, Bonjour
  • Local storage — file, database, app properties
  • Camera, GPS, accelerometer
  • Social APIs — Facebook, YQL, Twitter, more
  • Extensible & modular to add on custom functionality

Appcelerator Cloud Services

  • Add a server back-end without any server-side coding
  • Pre-built APIs for users, checkins, reviews, etc.
  • Photo storage — build your own Instagram!
  • Free tier and pay-as-you-go
You could use ACS to share data between users of your app -- say, to share tweet-like messages or Instagram-like photos.

You Can Build Apps For:

  • iOS — iPhone, iPad, iPod Touch
  • Android — handsets, tablets, Google TV
  • Mobile Web — HTML5 web apps from the same code
  • Blackberry and Windows Phone in the works
